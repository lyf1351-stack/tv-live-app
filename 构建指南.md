# TV Live App - 构建指南

## GitHub Actions 构建问题

React Native在GitHub Actions上构建非常复杂，需要：
- 大量配置文件
- 长时间构建（15-30分钟）
- 复杂的环境设置

## 推荐方案：Expo EAS Build

Expo Application Services (EAS) 是React Native官方的云构建服务，专为RN设计。

### 优势

✅ 专门为React Native优化
✅ 自动配置，无需复杂设置
✅ 构建快速（3-5分钟）
✅ 免费额度足够（每月15次构建）
✅ 自动上传到Artifact

### 使用步骤

#### 1. 安装EAS CLI

```bash
npm install -g eas-cli
```

#### 2. 配置项目

```bash
eas build:configure
```

#### 3. 登录Expo账号

```bash
eas login
```

#### 4. 构建APK

```bash
# 开发版本
eas build --platform android --profile preview

# 生产版本
eas build --platform android --profile production
```

### 构建类型

- **preview** - 内部测试APK（快速构建）
- **production** - 生产版本（带签名）

### 自动上传

构建完成后，APK会自动上传到：
1. Expo网站（可下载）
2. 本地Artifacts

## 备选方案：本地构建

如果不想使用EAS，可以本地构建：

```bash
cd android
./gradlew assembleRelease
```

APK位置：`android/app/build/outputs/apk/release/app-release.apk`
